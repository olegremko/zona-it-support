# Zona IT Backend

## Цель

Backend нужен, чтобы:
- хранить тикеты и сообщения в БД;
- привязывать все обращения к конкретной компании и пользователю;
- не давать клиентам видеть тикеты других компаний;
- вводить роли и матрицу прав для клиентов и сотрудников поддержки;
- вести аудит изменений.

## Роли

- `client_admin`: видит все тикеты своей компании, может создавать и обновлять их, управлять пользователями компании.
- `client_user`: видит и редактирует только свои тикеты, может создавать новые.
- `support_agent`: работает с тикетами назначенных компаний, отвечает клиентам.
- `support_lead`: распределяет заявки, видит больше служебных данных, пишет внутренние заметки.
- `platform_admin`: глобальный администратор платформы.

## Ключевые правила доступа

- Любой запрос к тикету проходит серверную проверку:
  - компания пользователя должна совпадать с `ticket.company_id`, либо пользователь глобальный админ;
  - если у пользователя нет `ticket.view.company`, он видит только свои тикеты;
  - внутренние заметки доступны только ролям с `ticket.comment.internal`.

## Следующий этап

1. Поднять runtime (`Node.js`) и установить зависимости.
2. Инициализировать БД через `npm run db:init`.
3. Заполнить демо-ролями и пользователями через `npm run db:seed`.
4. Перевести `zona-it-main.html`, `zona-it-portal.html`, `zona-it-chat.html` с `localStorage` на реальные `/api/*` запросы.
